# KMD 语言设计总纲

> 最近更新：2026-07-08
> 状态：封盘（规范定稿 / 大部分未实现，实现落差见 `migration.md`）

本文是 KMD 语言设计的总纲：定位、公理、词性总览、章节索引与封盘决议清单。
各子模型的完整定义在分章文档中，本文只记"是什么"和"定了什么"，不重复论证过程。

## 定位与原则

原则：可读性，正文和指令分离，针对创作和设计工作的最大易用性。

我们可能会将创作者变成脚本编写员，但不希望让作者在编写脚本时感到痛苦。
去写 kmd 脚本。不是使用者的话，不可能察觉到痛点所在。

KMD 不对标通用编程语言，它的参照系是：

- **CSS**：选择器 + 声明。作用范围指定是选择器问题。
- **GSAP / Motion Canvas**：时间轴 DSL。点链本质是一条微型时间轴。
- **LilyPond / ABC 记谱法**：正文与标注混排，领域符号密度高但领域内读者读得飞快。
- **Ink / Ren'Py**：叙事脚本 + 分支控制流。

加上控制流、变量后 KMD 会图灵完全，但**不以通用语言的方式组织**：
每一行都仍然是"一句演出"，新特性以"演出词汇"形态进入，而不是以"程序结构"形态进入。

## 五公理

所有新特性设计必须逐条过这五关：

1. **正文优先**。正文在左，`@` 之后是谓语。遮住 `@` 右边，剩下的仍是一篇可读的文章。
2. **链即时间**。点链从左到右就是时间从先到后。
3. **符号管时序，单词管语义**。`> ~ ^ | ! -> ~>` 是时序/流向符号，`red wave goto` 是语义单词，永不互串。
4. **后缀管修饰**。`:unit` 管实例化粒度，`!` 管阻塞，后缀不改变词的身份。
5. **双钟分立，桥接显式**。正文播放时钟与链内时钟互不隐式影响；任何跨钟作用必须有可见语法（粒度后缀 / `pause` / `!`）。

## 词性总览

一条 KMD 指令是一个句子，词性有四种（完整定义见 `chain-model.md`）：

| 词性 | 位置 | 例 | 回答的问题 |
| --- | --- | --- | --- |
| 主语 | 链头 | `{真相}` `cam` `flow` `upobj` 裸 `.` | 作用于谁 |
| 谓语 | 链身 | `red` `shake(5)` `goto(p)` `(cam.zoom(0.1))` | 发生什么 |
| 连接符 | 谓语之间 | `-1s->` `~2s~>` | 何时/如何过渡 |
| 引用 | 谓语位 | `$name` `$(expr)` | 展开存好的链 |

另有一类**量词字面量**（`1s` `0.5line` `+=0.1` `3~1s~>0`），作为参数值存在，见 `chain-model.md`。

## 封盘决议清单

> 2026-07 语言设计收敛讨论（两轮）的全部决议。编号供分章与迁移文档引用（D1–D27）。

| # | 决议 | 所属分章 |
| --- | --- | --- |
| D1 | 主谓模型：链头是主语（receiver），统一走一次作用域链查找；`cam`/`var`/`flow` 是内建主语，用户对象是同类居民 | chain / scope |
| D2 | 值语义 vs 引用语义：宏 = 存了链片段的变量（值，展开即拷贝）；对象有身份（引用，追溯性修改）。声明统一用 `=`，差别由所存内容和用法决定 | chain / scope |
| D3 | 卫生宏：宏内时序标记只在宏自身局部时间轴生效，展开点将宏视为原子成员，并发合法 | chain |
| D4 | `$` 取值展开：`$(expr)` 在谓语位求值出一段链并原位拼接；`$name` 是 `$(name)` 简写；构建期求值；省略 else = 空链。特效控制流由此涌现，无新机制 | chain / control-flow |
| D5 | `f` 废除。token 选择器由 `{}`（顺序）与 `{文本}`（内容）接任，`@` 后的花括号主语与正文括号组视觉镜像 | selection |
| D6 | 对象成员操作：`+eff` 追加、`-eff` 删除、`+eff(params)` 已有则合并参数；默认（无符号）为整链替换 | chain |
| D7 | 值域字面量 `a~>b` / `a~d~>b`（成员局部 tween）；相对值 `+=` / `-=`（弃用裸 `+0.1`） | chain |
| D8 | 连接符：`-d->` 保持后切换，`~d~>` 缓动过渡（缓动曲线作连接符参数 `~d,powerIn~>`）；时序成员不再以单词谓语形态存在 | chain |
| D9 | 拍（beat）：连接符切分拍；同拍谓语在拍首同时起跑；参数值域是拍内局部动画。糖与全写是两种不同动画语义，非繁简关系 | chain |
| D10 | 从句：`.(主语.谓语链)` 在谓语位嵌入带自身主语的完整句子；`camZoom` 等合成词降级为历史糖 | chain |
| D11 | 从句阻塞标记外置为规范形 `.(...)!`，内置写法等价接受，格式化归一到外置；默认发射后不管 | chain |
| D12 | 实例化粒度 `:char/:group/:block`，读法"逐字/逐组/逐段"：被修饰成员按 unit 实例化，每实例对齐该 unit 揭示时刻；只约束单一成员（含从句），从句是打包工具；触发堆积由 mutex group 裁决 | chain |
| D13 | 双钟不变量（公理 5） | 总纲 |
| D14 | 选择器双类型：点（零宽位置）与域（文字区间），访问器互转（`.line` `.start` `.end` `range(a,b)`）；谓语声明所需类型，不匹配走诊断 | selection |
| D15 | `:::name` 围栏声明多行选区；围栏名 = 预绑定选区的用户对象（选择器与响应式对象是同一种东西）；围栏行上只许有名字 | selection |
| D16 | 内容选择器支持前缀模糊匹配；歧义即诊断报错，绝不静默取首个；`{文本:2}` 序号消歧。裸文本（非括号组）匹配留二期 | selection |
| D17 | 链数与括号组数失配 = 诊断错误，废除 first/last 静默重分配 | selection |
| D18 | 裸主语排版链（`@ up(100)`）弃用，归入裸点（`@ .up(100)`）；`flow` 主语留给显式游标操纵 | selection / migration |
| D19 | 作用域链统一：场景级用户定义 → 文档级定义 → 内建主语 → 特效/指令名；`var` 是文档级作用域的名字，`var.x` 为全限定形式；`@ var.x = ...` 随处声明文档级；裸名 `@ x = ...` 为场景级 | scope |
| D20 | 生命周期 = 脚本文本区间（定义处到所属场景 `---`；文档级全程）；播放状态是脚本位置的纯函数，回翻/seek 恢复对象 | scope |
| D21 | 全局命名冲突律：禁 shadowing；用户名冲突、跨注册表指令重名均为诊断 | scope |
| D22 | 选项级联：frontmatter = 文档级选项，`[key=value]` = 段级选项（不粘滞），就近取值；引擎启动键（`mode` 等）文档级专属；粘滞语义交给变量，选项保持无状态。`[...]` = 段级选项 + 段首句子（含省略主语的 `.glitch` 与 `[if]`） | scope |
| D23 | 控制流三高度：行内 `{=cond ? A \| B}`（文字变体）、段落 `[if cond]`（有无）、结构 `[if cond -> #标签]`（分支）；`#` 标题即跳转锚点；`{=}` 是表达式括号的唯一入口，正文裸括号恒为文字 | control-flow |
| D24 | 量词字面量立为独立字面量类：时间（`s/ms`）、空间（`char/line/self/px`）、角度（`deg`）、相对（`+=/-=`）、值域（`~>`），解析器给类型化表示 | chain |
| D25 | `loop` 不存在：后向导向即循环。跳回脚本序更早的锚点时，场景级对象按位置重建（D20）、文档级 `var` 跨跳转存活（计数器天然实现）；回边由 builder 按脚本序自动检测，无专门语法 | control-flow |
| D26 | `->` 的独立形态是结构行（行首独占一行，无条件导向），与 `---`/`#`/`:::`/`[...]` 同族；条件形态住块 if；不存在 `@ ->`（与 `@ if` 不存在同一裁决） | control-flow |
| D27 | `wait` 不存在："停到外部事件"是时长由事件决定的 pause。pause 时长参数接受事件类量词（`pause(click)`；正文糖 `\|(click)`）；`signal:name` 留 Phase C。量词字面量增"事件"类。响应式对象与此正交（对象管"变什么"，pause 管"何时"） | control-flow / chain |

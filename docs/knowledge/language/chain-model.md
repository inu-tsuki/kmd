# 链语法模型

> 最近更新：2026-07-08
> 状态：封盘（规范定稿 / 未实现）
> 决议依据：D1–D4、D6–D13、D24（见 `design.md` 封盘清单）

一条 KMD 指令是一个**句子**。本章定义句子的完整语法：四种词性、拍、从句、实例化粒度、字面量与取值展开。

## 句子模型

```
句子 := 主语 . 谓语序列
谓语序列 := 拍 (连接符 拍)*
拍 := 谓语成员+（同拍成员以 . 相连）
谓语成员 := 指令名[:粒度][(参数)][!] | 从句 | 引用
从句 := ( 句子 )[:粒度][!]
引用 := $名字 | $(表达式)
```

多个句子可写在同一 `@` 之后：**空白分隔**表示各配各的选区槽位（见 `selection-model.md`），**`+` 连接**表示并联（同时起跑的两个完整句子）。

## 主语（链头）

链头永远是主语，回答"这条链作用于谁"。主语通过统一的作用域链查找（见 `scope-and-lifetime.md`），种类包括：

- 选择器：`{}`、`{文本}`、裸 `.`（本行/本段）、`mark(a)`、`prev`、`next` 等，见 `selection-model.md`；
- 内建对象：`cam`、`flow`、`var`；
- 用户对象：围栏名、`@ name = ...` 定义的名字。

主语永远在头。要在链身**引用**存好的链，用 `$`（见下文），不存在 `f.effobj` 或 `effobj.f` 之类的角色混杂形态。

## 谓语与拍

谓语是作用在画布上的动词：`red`、`shake(5)`、`goto(p)`、从句。

**拍（beat）**：连接符切分拍。**同一拍内的谓语全部在拍首同时起跑**；参数值域（`a~d~>b`）是该谓语自己的拍内动画，时长自带、互不阻塞。

```kmd
{它}.shake(3~1s~>0).wave(5~2s~>0)          // 一拍：两个 tween 同起点、各自时长
{它}.shake(3).wave(5) ~1s~> shake(0).wave(0)  // 两拍：所有属性锁定同一条 1s 过渡
```

两种写法是**两种不同的动画**（各自呼吸 vs 整体转场），不是繁简关系。

## 连接符

时序不以单词谓语形态存在（公理 3），一律写作谓语之间的连接符：

| 形态 | 语义 |
| --- | --- |
| `-d->` | 保持 d 时长后切换（原 `hold(d)`） |
| `~d~>` | d 时长的缓动过渡（原 `ease(d)`） |
| `~d,curve~>` | 指定缓动曲线，如 `~1s,powerIn~>` |

```kmd
{准备}.red -1s-> blue          // 红，保持 1s，瞬切蓝
```

## 从句

谓语位可以嵌入一个带自身主语的完整句子，用括号打包：

```kmd
你说什么？ @ .pause:char(1s).(cam.zoom(+=0.01)):char
cut them. @ upobj.powerIn(0.5s).left(0.5line) + downobj.right(0.5line)
```

- 从句主语可以是任何主语（`cam`、`flow`、用户对象……）。舞台/排版/对象指令进链的通路由此统一，`camZoom` 等合成词降级为历史糖。
- 默认**发射后不管**（不占链时间）；挂 `!` 则阻塞链的下一拍。
- `!` 规范形**外置**：`.(cam.zoom(2s))!`。内置写法 `.(cam.zoom(2s)!)` 等价接受，格式化工具归一到外置。

## 实例化粒度（`:unit`）

`:char` / `:group` / `:block` 后缀，读法"**逐字 / 逐组 / 逐段**"：

> 被修饰的成员按 unit 实例化，每个实例对齐该 unit 的揭示时刻。

- `wave:char` = 逐字一个 wave 实例（错相波动）；
- `pause:char(1s)` = 逐字停 1s；
- `(cam.zoom(+=0.01)):char` = 逐字触发一次从句。

"作用粒度"与"触发节拍"是同一机制在空间型/时间型成员上的不同观感。**覆盖范围永远不归 `:` 管，归主语管**。

约束规则：粒度**只约束它所依附的单一成员**（含从句），不传染后续链。想让一串成员整体逐字重复，用从句打包再挂后缀：

```kmd
@ .(pause(1s).(cam.zoom(+=0.01))):char
```

每次触发是一次全新实例；触发堆积由 effect mutex group 裁决。值域与粒度的交互：`(cam.zoom(0.7~2s~>1.0)):char` = 每次触发重启一段 2s tween。

## 字面量

### 量词

量词是独立的字面量类，解析器必须给出类型化表示（不得退化为字符串）：

| 类 | 形态 |
| --- | --- |
| 时间 | `1s` `500ms` |
| 空间 | `1char` `0.5line` `1self` `100px`（裸数字按指令约定的默认单位） |
| 角度 | `15deg` |
| 速率 | `15deg/char` `0.98/char`（每 unit 变化量，用于 flow 等） |
| 相对 | `+=0.1` `-=0.1`（相对当前值；裸 `+0.1` 即正数，不再有相对语义） |
| 值域 | `a~>b`（默认时长）`a~d~>b`（指定时长），成员局部 tween |
| 事件 | `click`（`signal:name` 留 Phase C）。作时长参数时表示"停到事件发生"：`pause(click)`、正文糖 `\|(click)`，见 `control-flow.md`（D27） |

### 表达式

链内参数位天然是表达式语境，可直接使用变量与条件：

```kmd
{真相}.shake(strength = var.trust > 3 ? 20 : 5)
```

## 取值展开（`$`）

`$(expr)` 在谓语位求值出一段链并**原位拼接**；`$name` 是 `$(name)` 的简写。构建期求值（与场景烘焙时机吻合）；条件缺 else 分支 = 空链。

```kmd
{红绿灯} @ {}.$trafficLight.yellow
{真相} @ {真相}.$(var.route == 'a' ? red.shake : gray).wave
{线索} @ {线索}.$(var.found ? highlight)
```

这就是特效控制流的全部机制：参数条件（改强度）、`$()` 拼接（换特效）、`[if]` 块选项（整段有无，见 `control-flow.md`）。

## 宏与对象操作

**宏 = 存了链片段的变量**（值语义）：使用处展开为独立拷贝，无身份。
**对象**（围栏名、被文字应用过的名字）有身份（引用语义）：修改追溯性地作用于其覆盖的全部文字。

卫生性：宏内的时序标记只在宏自身的局部时间轴生效，展开点将整个宏视为一个原子成员，因此宏参与并发是合法的。

对象成员操作：

```kmd
@ effobj.+wave              // 追加
@ effobj.-wave              // 删除
@ effobj.+shake(strength=5) // 已有 shake 则合并参数
@ effobj.rainbow            // 无符号：整链替换
```

## 实现落差

- `KMDCommandParser.parseEffectChain` 目前用字符串替换保护 `cam.`（占位符黑客），从句语法落地后由括号深度计数根治。
- 成员解析正则不支持嵌套括号，从句 / `$()` / 值域 / `+=` 均要求升级为小型递归下降解析。
- `autoConvert` 仅处理 `s/ms`，其余量词退化为字符串；需按 D24 类型化。

# 选区与流模型

> 最近更新：2026-07-08
> 状态：封盘（规范定稿 / 未实现）
> 决议依据：D5、D14–D18（见 `design.md` 封盘清单）

本章定义"特效罩住谁"：选择器的类型系统、选择器家族、多行选区（围栏）与排版流主语。

## 点与域：选择器的双类型

选择器有两个类型：

- **点**：零宽位置（标记点、域端点）；
- **域**：一段文字区间（括号组、行、围栏区域）。

访问器在两者间转换，选择器代数由组合闭合：

| 转换 | 形态 |
| --- | --- |
| 点 → 域 | `mark(a).line`（扩到所在行；未来 `.para`） |
| 域 → 点 | `{真相}.start` / `{真相}.end`（`prev.start`、`next.end` 同族） |
| 点 + 点 → 域 | `range(a, b)` |

**谓语声明所需类型**：`goto` 要点，视觉特效要域。类型不匹配走诊断（"goto 需要一个点，你给了一个域，要 `.start` 还是 `.end`？"）。

## 选择器家族

| 选择器 | 类型 | 语义 |
| --- | --- | --- |
| `{}` | 域 | 顺序绑定：第 N 条链配本行第 N 个括号组（原 `f` 的行为） |
| `{文本}` | 域 | 内容绑定：命中正文中括号组 `{文本}` |
| 裸 `.` | 域 | 本行；空正文行时为本段 |
| `mark(a)` | 点 | 命名标记点 |
| `prev` / `next` | 域 | 相邻行（挂 `.start`/`.end` 取点） |
| `range(a, b)` | 域 | 两点之间（可跨段） |
| 围栏名 | 域 | `:::name` 声明的多行区域（见下） |

规则：

- **顺序与内容可混用**；同一 token 可被多条链选中（`{真相}.red.shake  {真相}.(cam.zoom(0.1))`）。
- **内容绑定支持前缀模糊匹配**（`{真}` 命中唯一一个以"真"开头的组）；**歧义即诊断报错**，绝不静默取首个；同文本重复用序号消歧 `{真相:2}`。
- **链数与括号组数失配 = 诊断错误**。想广播用裸 `.`，想精确指定用 `{文本}`。（废除旧的 first/last 静默重分配。）
- 选择器悬空（正文改动后内容绑定失配）走诊断，不静默忽略。
- 裸文本匹配（选中未加括号的正文）是二期愿景，见 `brainstorm.md`；与模糊匹配目标不同（书写便利 vs 表达力扩张），互不许诺。
- 裸主语排版链（`@ up(100)`）已弃用，写作裸点 `@ .up(100)`（排版指令作用的本来就是"这一行在流里的位置"）。

## 围栏：多行选区

多行选区必须显式定界（markdown 容器指令血统）：

```kmd
:::upobj
text text text text text text
text text text text text text
:::

cut them. @ upobj.powerIn(0.5s).left(0.5line).up(1char) + downobj.right(0.5line)
```

- **围栏行上只许有名字**（`:::name`），不许挂选项链。
- 围栏可嵌套、可折叠；与 `---`、`[...]`、空行 `@` 同属"非正文行"家族，单行/单段作用域永远不需要围栏。
- **围栏名 = 预绑定了选区的用户对象**：拿它当选择器（`upobj.left(...)`），也拿它当响应式对象（`upobj.+wave`）——选择器与对象在语言里是同一种东西（一个指称了某片文字、能承载特效的主语）。
- 重名冲突走诊断（禁 shadowing）；生命周期见 `scope-and-lifetime.md`。

## 流主语（`flow`）：异形排版

排版流是一个有位置、方向、缩放的**游标**；异形排版 = 操纵游标，不需要新概念。

低阶词汇——游标谓语（速率量词 `x/char` 逐字微调）：

```kmd
:::spiral [flow]
回转的文字沿着螺旋一路旋转下去…… @ flow.turn(15deg/char).scale(0.98/char)
:::
```

回转、波浪基线、放射排版全是这一个机制。

高阶词汇——把流绑到锚点/方向上（分支排版）：

```kmd
主干文字向上生长 @ flow.dir(up).mark(fork)
:::branchL [flow.from(fork).dir(upleft)]
左枝的文字
:::
:::branchR [flow.from(fork).dir(upright)]
右枝的文字
:::
```

树 = 多条流共享一个锚点。`mark` 复用定位系统，`:::` 复用围栏——异形排版只新增了 `flow` 这一个主语。
（螺旋 / 文字树 / 斩断文字流三个期望 demo 记录在 `brainstorm.md`。）

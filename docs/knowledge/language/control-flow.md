# 控制流

> 最近更新：2026-07-08
> 状态：封盘（规范定稿 / 未实现）
> 决议依据：D4、D23、D25–D27（见 `design.md` 封盘清单）

本章定义正文播放的控制流（三高度 if、锚点跳转、行内表达式）与特效控制流。
参照系是 Ink / Ren'Py：分支即"命名段落 + 跳转"，而非通用语言的语句块。

## 三高度 if

同一个 `if` 词汇，三个语法位置，各管各的语义强度：

| 高度 | 形态 | 语义 |
| --- | --- | --- |
| 行内 | `{=cond ? A \| B}` | 一行内的文字变体 |
| 段落 | `[if cond]` | 整段的有无（不改流向） |
| 结构 | `[if cond -> #标签 \| ...]` | 分支（改流向） |

单独一行想条件显隐：让它自成一段挂 `[if]`——段落切分本身是 KMD 的表达手段，不为"段内单行"另立语法。

## 表达式括号 `{=}`

正文里裸括号组恒为文字（`{红绿灯}` 是字）；**进入表达式必须写 `{=`**：

```kmd
你好，{=var.name}。
{=var.mood ? happy: 她笑了 | sad: 她垂下眼 | 她沉默着}。
```

- 插值与条件共用一个标记；多分支复用 `条件: 文本 |` 形态，与段落 if 同形异位。
- 行内变体放正文是有意为之（Ink 判例：叙事变体的局部性压倒纯净性——作者必须在变体发生处读到所有变体）。
- 纪律：`{=}` 里只许"条件 + 字面文本"；更复杂的逻辑（计算、多级嵌套）上浮到 `@` 行或 frontmatter。超过一两个分支建议上浮（风格建议，非硬规则）。

## 段落有无：`[if]`

```kmd
[if var.trust > 3]
她犹豫了一下，还是把信递给了你。
```

条件不满足 = 该段不进时间轴，播放直接流过。

## 结构分支：`#` 锚点 + `->` 导向

`#` 标题（现有 markdown 糖）同时是**文档结构**（可渲染、大纲可导航）和**跳转锚点**——章节本来就是天然的跳转目标：

```kmd
[if var.trust > 3 -> #真相 | else -> #谎言]

# 真相
她终于说出了一切……

# 谎言
"没什么。"她移开了视线。
```

`->` 表"导向"，与值域 `~>`、连接符 `-d->` 同为流向符号家族（公理 3）。
跳转与 seek 语义天然兼容：播放状态是脚本位置的纯函数（见 `scope-and-lifetime.md`），任何导向等价于一次受控 seek。

## 独立导向与后向跳转

`->` 的独立形态是**结构行**（与 `---`、`# 标题`、`:::`、`[...]` 同族）：行首 `->`，独占一行，无条件导向：

```kmd
她转身离开了。

-> #第二幕
```

条件形态住块 if。**不存在 `@ ->` 形态**——`@` 句子管"本行正文怎么演"，导向管"文档往哪走"，词不配位（与 `@ if` 不存在是同一裁决）。

**后向导向即循环（D25）**：不存在 `loop` 语法。跳回脚本序更早的锚点时：

- 场景级对象按脚本位置重建（D20 纯函数原则）——循环体第二次进入是干净的；
- 文档级 `var` 跨跳转存活——计数器自然实现 while 循环：

```kmd
# 副歌
啦啦啦……
@ var.i = var.i + 1
[if var.i < 3 -> #副歌]
```

运行时：`SegmentGraphBuilder` 检测跳转目标在脚本序更早的位置，自动将该边标为回边（back-edge），复用入口 checkpoint 恢复机制（见 `segment-graph-runtime-draft.md` §5.2，counter 由用户 `var` 承担）。

## 事件时长暂停（wait 之死）

**不存在 `wait` 语法（D27）**。"停到外部事件发生"是**时长由事件而非时钟决定的 pause**——pause 的时长参数接受事件类量词（见 `chain-model.md` 量词表）：

```kmd
@ pause(click)                  // 指令形：停到点击
真相是……|(click)我骗了你。      // 糖形：正文停顿符的事件版（视觉小说式点击继续）
```

- 事件类量词一期只有 `click`；`signal:name` 留 Phase C（SignalRegistry）。
- 运行时编译为 wait-gate 边（段边界处）或 timeline 内 gate（行中），播放相位 `waiting`，形态见 `segment-graph-runtime-draft.md` §5.4。
- 响应式对象与此正交：对象管显示"变什么"（空间/内容），pause 管播放"何时继续"（时间）——公理 5 双钟分立。

## 特效控制流

不引入新机制，由表达式与 `$()` 涌现（详见 `chain-model.md`）：

```kmd
{真相}.shake(strength = var.trust > 3 ? 20 : 5)     // 参数级：改强度
{真相}.$(var.route == 'a' ? red.shake : gray).wave  // 成员级：换特效
{线索}.$(var.found ? highlight)                      // 缺 else = 空链
```

整段特效的有无由 `[if]` 承担。三个火力等级覆盖"段落控制 + 指令单句 + 响应式变量"原本要拼装才能做到的事。

`$()` 为构建期求值；播放期动态改变走响应式对象通道（`effobj.+wave` 等引用语义操作），两者不打架。

## 实现落差

- 分支跳转要求 `ScriptPlayer` / `SegmentBuilder` 支持段落图（非线性 segment 序列）；与 `docs/planning` 中 Phase B 控制流规划（SegmentGraphPlan）对接。
- 行内 `{=}` 要求括号组解析在扫描期区分表达式括号与文字括号。
- `[if]` 复用块选项通道，条件求值时机与场景烘焙对齐。

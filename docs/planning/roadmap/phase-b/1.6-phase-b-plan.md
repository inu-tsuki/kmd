# KMD 1.6 Phase B Feature Plan

> 状态：Roadmap / gated by integration stability（语言收敛审查已于 2026-07-08 完成）
> 最近更新：2026-07-08
> 范围：链语法模型落地、作用域/StateStore、Document Control Flow、Segment Graph，以及随功能推进同步偿还的 execution 技术债
>
> 语言规范权威：`docs/knowledge/language/design.md`（五公理 + 封盘决议 D1–D27）及其分章
> （`chain-model.md` / `selection-model.md` / `scope-and-lifetime.md` / `control-flow.md`）。
> 本计划引用决议时使用 D 编号；旧→新语法对照与解析器工程债清单见 `docs/knowledge/language/migration.md`。
>
> 运行时形态补充：`segment-graph-runtime-draft.md`（草案 / 未立项）——B3/B4 占位词的数据结构与边求值规则。
> 本计划管"做什么/什么顺序"，那份管"运行时长什么样"。

## 1. 定位

Phase A 与 Phase B Prep 已经完成 parser、layout、execution、stage、diagnostics 和 metadata 的主链路准备。
2026-07 的语言设计收敛（D1–D27）把 Phase B 曾经悬而未决的语言问题全部裁定：主谓模型、选择器家族、
连接符与拍、从句、实例化粒度、两级作用域、三高度控制流。Phase B 的任务从"边做边定语法"变为
**"按封盘规范实施"**。

Phase B 引入的用户可见能力（以新规范词汇表述）：

- 新链语法：主语作用域链、`{}`/`{文本}` 选择器、连接符与拍、从句、实例化粒度、量词字面量（D1、D5、D8–D12、D24）
- 宏/对象与 `$()` 取值展开、`:::` 围栏选区（D2–D4、D6、D15）
- 两级作用域与 StateStore、`{=}` 表达式（D19–D22）
- 三高度控制流：`[if]`、`[if -> #标签]`、`#` 锚点、`->` 独立结构行、行内 `{=cond ? A | B}`（D23、D25–D26）
- 事件时长暂停 `pause(click)` / 正文糖 `|(click)`（D27）
- `SegmentGraphPlan` 与跨 Segment seek
- 跨 `.kmd` 文档引用保持为设计入口（brainstorm，不实施）

每个功能工作包仍必须绑定一个债务收口目标，避免 `TextPlayer`、`SegmentBuilder`、`EffectProcessor`
和 core/store 耦合继续膨胀。

Phase B 恢复条件：

- reader-runtime-web artifact 消费链路在 Android 真机/模拟器 smoke 中稳定。
- Phase B 实施不引入 Vue、Pinia、Monaco、TextMate 或 editor panels 到 `@kmd/reader-runtime-web` 边界内。
- ~~语言设计收敛审查~~ **已完成**（2026-07-08，产出 D1–D27 与 `knowledge/language/` 总-分文档）。

## 2. 设计原则

- 新语法先进入 `DocumentAST / DocumentSemanticIR`，不要直接投影成 legacy token 或 layout stream item。
- Segment 内部仍是确定性 `gsap.Timeline`；Segment 之间的边由 graph runtime 按 state/control-flow 求值。
- `LayoutStream` 只作为 layout lane 内部表示，不承载 stage、playback、control-flow 总线职责。
- 新 execution 语义必须进入 plan/cue，不新增 legacy `TextPlayer.play()` 分支。
- core 层不新增 `useEditorStore()`，后续改为 host callback / adapter。
- stage/effect/layout 行为优先读 metadata，不新增硬编码 command set。
- 新增（语言收敛后）：实施顺序遵循 `migration.md` 的迁移路径——先兼容投影保行为不变，再开诊断，再迁语料，最后弃用旧形态。
- 新增：所有静默裁决路径按 D17/D21 转诊断（失配重分配、跨注册表重名），不保留无诊断的启发式。

## 3. 目标管线

```text
Source
  -> DocumentParser
  -> DocumentAST
  -> AstNormalizer
  -> DocumentDependencyResolver
  -> SemanticLowerer / ControlFlowLowerer / StateLowerer
  -> DocumentSemanticIR
  -> EffectMiddleware / LayoutMiddleware / StageMiddleware
  -> ParagraphExecutionPlan
  -> SegmentExecutionPlan
  -> SegmentGraphPlan
  -> GraphPlaybackRuntime
```

第一轮不要求一次落齐所有层。实现上可以保留当前 `ParagraphIR` 与 compat projection，但 Phase B
新能力必须有 document-level 语义节点，不再伪装成段落内普通命令。

## 4. 工作包

### B0. Chain Syntax Frontend（重写自"Syntax Frontend"）

按封盘规范重建链解析。分四个子包，B0.1 行为中立，之后逐步开语义。

**B0.1 新链解析器与量词类型化**（D24；migration 工程债 1/2/6/7）

- 成员解析从正则升级为小型递归下降：支持嵌套括号、从句、`$()`、值域、连接符。
- 链拆分回归纯括号深度计数，删除 `NAMESPACE_CAM_DOT_` 占位符黑客。
- 量词字面量类型化（时间/空间/角度/速率/相对/值域），不再退化为字符串。
- 旧形态全部走 `CompatProjector` 投影，行为零变化。

**B0.2 主语作用域链与选择器**（D1、D5、D16–D18、D21）

- 链头统一作用域链查找（场景级 → 文档级 → 内建主语 → 指令名），`ScopeRouter` 的 `"f" | "dot" | other` 三分法退役。
- `{}` 顺序选择器与 `{文本}` 内容选择器（前缀模糊匹配，歧义即诊断）；`f.` 转兼容形。
- 链数/括号组数失配转诊断，废除 first/last 静默重分配（**行为变更**，语料需同步迁移）。
- 裸排版链（`@ up(100)`）弃用诊断，归入裸点。
- 跨注册表重名转诊断。
- 点/域双类型与访问器（`.line` / `.start` / `.end` / `range`）第一批落地（D14）。

**B0.3 连接符、拍、从句、粒度**（D8–D12）

- 连接符 `-d->` / `~d~>` 与拍模型进入 AST 与 execution plan；`hold`/`ease` 单词形态转兼容投影。
- 从句 `.(主语.谓语链)[!]` 进链，`camZoom` 等合成词降级为历史糖。
- 实例化粒度 `:unit` 按"单成员约束 + 从句打包"重述；`pause:char` 行为兼容、语义澄清。
- `+` 并联句子表达为 `ParallelSentenceAst`，不在 `EffectProcessor` 中二次猜测。
- 行尾 `\` 续行归入 `AstNormalizer` 或 document parse 预处理。

**B0.4 宏、对象与围栏**（D2–D4、D6、D15）

- `@ name = 链` 定义（值语义）与 `$name` / `$(expr)` 取值展开（构建期求值）；卫生宏（宏内时序局部时间轴）。
- 对象引用语义与 `+eff` / `-eff` / `+eff(params)` 成员操作。
- `:::name` 围栏选区进 DocumentAST（围栏名 = 预绑定选区的对象；围栏行只许名字）。
- 宏/对象引用生成结构化 `ReferenceNode` / `MacroDefinitionNode`，不把用户名字伪装成普通 effect name。
- 跨 `.kmd` 引用保持设计入口：允许声明依赖，首轮不实现 runtime 自由加载（依赖必须过 `DocumentDependencyResolver`）。

验收（B0 全体）：

- parser fixture 覆盖各子包新语法；每步 `pnpm test:parser` 通过。
- compat projection 对未启用新语法的脚本零行为变化（B0.2 的失配诊断除外，语料先迁）。
- `apps/editor/public/tests/` 语料按 `migration.md` 步骤 3 逐个迁到新形态，作为回归样本。

### B1. Scope, StateStore And Expressions（重写自"StateStore And Expressions"）

功能（D19–D22）：

- `StateStore { get, set, snapshot, restore }` 作为**两级作用域（文档级 var / 场景级）的运行时载体**。
  旧 open question "state.* 与 var.* 双命名空间"已由 D19 裁决：不存在独立 state 命名空间，
  `var` 就是文档级作用域，裸名是场景级。
- 极简表达式求值：字面量、变量、算术、比较、逻辑、`?:` 条件。链内参数位与 `{=}` 共用同一求值器。
- `@ var.gold = var.gold + 5`（文档级）与 `@ temp = ...`(场景级) 赋值语句。
- 生命周期 = 脚本区间（D20）：`Checkpoint` 增加 `state`，回翻/seek 按脚本位置恢复作用域内容。
- 选项级联（D22）：frontmatter/块选项收编为同一张选项表的两级，就近取值；引擎启动键段级出现走诊断。

偿债：

- `RuntimeValueResolver` 成为作用域链的统一解析入口（含 marker）。
- 当前 frontmatter `variables` 写入 `layout.globalMarkers` 的路径收口为兼容投影，避免作用域与 marker 双真相。
- `autoConvert` 的 `var.` 字符串直通特判删除。

验收：

- seek 前后作用域内容可恢复；页模式回翻对象不丢（D20 场景验证）。
- 赋值不依赖 stage preset 伪装执行。

### B2. Document Control Flow（表面语法按 D23 重述）

功能：

- `DocumentParser` / document-level parse entry：段落 parser 不再负责文档结构；围栏、`#` 锚点、块选项句子在这一层结构化。
- 三高度 if（D23）：
  - 行内 `{=cond ? A | B}`（含多分支 `{=x ? a: A | b: B | C}`）——文字变体，依赖 B1 求值器；
    扫描期区分表达式括号与文字括号。
  - `[if cond]`——段落有无（不改流向）。
  - `[if cond -> #标签 | else -> #标签]`——结构分支；`#` 标题即跳转锚点，`->` 即导向。
- 旧 `@ tag` 由 `#` 标题取代；旧 `@ jump target` 由 `->` 取代——独立形态为结构行（行首 `-> #标签` 独占一行，D26），条件形态住块 if。
- 旧 `@ loop` 取消（D25）：后向导向即循环，`SegmentGraphBuilder` 按脚本序自动检测回边，counter 由用户 `var` 承担。
- 旧 `@ wait` 取消（D27）：`pause(click)` 指令形与 `|(click)` 正文糖；运行时仍编译为 wait-gate 边 / timeline 内 gate（`waiting` 相位）；`signal:name` 留 Phase C。

偿债：

- `ControlFlowLowerer` 输出结构化 control nodes，不把控制流压进 `paragraph.globalEffects`。
- 控制流 diagnostics（未定义锚点、重复锚点、不可达分支）进入统一 collector，build 期报。
- 跨文档跳转先作 graph edge metadata，不直接触发 loader 行为。

验收：

- 能生成 document-level control tree（含 `#` 锚点索引）。
- 第一阶段可只支持 syntax + graph plan，不急于完整 runtime 行为。

### B3. SegmentGraphPlan

（与收敛前一致，运行时形态见 `segment-graph-runtime-draft.md`；仅词汇更新：tag = `#` 锚点。）

功能：

- `SegmentGraph { nodes, edges, defaultPath }`。
- 每个 graph node 持有一个烘焙好的 deterministic segment。
- edges 支持 condition、default、jump target（`#` 锚点）、wait gate。
- 未来 edges 可以持有 `documentId` / `moduleId`，表达跨 `.kmd` 边界。

偿债：

- `SegmentBuilder` 降级为"单 Segment builder"，不再承担 graph 切割。
- 抽出 `SegmentGraphBuilder`，负责 paragraph group 切割、edge 生成、锚点索引。
- stage tween conflict 与 in-flight record 继续保留，但 planner 先产出 diagnostics，runtime trim 只做兜底。

验收：

- 线性脚本能被表示为只有 default edges 的 graph。
- 原有 single-segment playback 行为保持一致。

### B4. Graph Playback And Seek

（与收敛前一致。）

功能：

- 同 Segment 内 seek 继续使用 `timeline.seek(localTime)`。
- 跨 Segment seek 恢复目标 `entryCheckpoint`（含 B1 的 `state`），再 seek 到本地时间。
- 条件分支 seek 默认沿 `defaultPath`。
- loop/while seek 先采用入口 checkpoint + warp replay（表面语法待定不影响该机制）。

偿债：

- 抽出 `GraphPlaybackController`，不要继续把 graph 行为塞进 `ScriptPlayer`。
- `ScriptPlayer` 收缩为 load/build/play facade。
- core 到 editor store 的时间、行号、marker 更新改为 callbacks。

验收：

- seek、play、pause API 对 editor 仍稳定。
- 新 graph runtime 不调用 legacy `TextPlayer.play()`。

### B5. Execution Debt Closure

（与收敛前一致。）

任务：

- 标记 legacy `TextPlayer.play()` 为不再扩展路径；达到覆盖条件后删除或移到 compat module。
- 将 `TextPlayer.buildTimeline()` 中剩余 chain 分流继续前移到 execution plan / EffectMiddleware。
- 拆 `SegmentBuilder.applyStageConfigs()` 为 stage timeline coordinator。
- `KineticChar` 不新增语义字段，逐步只保留 display host 与 transform state。

验收：

- 新能力只消费 `ParagraphDisplayAssembly`、execution plan、graph plan。
- `KineticChar.stageInstructions / visualEffects / timingSugars` 仍仅作为 deprecated mirror。

### B6. Editor And Diagnostics Integration

功能：

- Monaco 控制流折叠（围栏、`#` 锚点）与 segment boundary 标记；`packages/language` 语法资产随新语法更新（`pnpm language:check`）。
- timeline markers 展示 graph default path。
- Inspector 显示 command metadata、作用域 snapshot、graph edge diagnostics。
  （Pharo 式活性 inspector 是长线愿景，见 `knowledge/language/brainstorm.md`，本包只做快照展示。）
- doesNotUnderstand 风格诊断：链头/指令名查找失败给出建议（D19 配套）。

偿债：

- editor UI 通过 adapter 读取 runtime snapshot，不让 core import Pinia。
- diagnostics/audit 继续走统一总线，不新增孤立 `console.log` 调试路径。

验收：

- Phase B 新 diagnostics 能在 editor 和 console sink 中同时观察。

## 5. 推荐实施顺序

1. B0.1 新链解析器 + 量词类型化（行为中立，compat 投影全绿）。
2. B0.2 主语作用域链 + 选择器 + 诊断开闸；同步迁移 `public/tests/` 语料。
3. B0.3 连接符/拍/从句/粒度。
4. B1 作用域 StateStore + 表达式 + `Checkpoint.state` + 选项级联。
5. B0.4 宏/对象/围栏（`$()` 依赖 B1 求值器，故排在其后）。
6. B2 document control-flow syntax tree（三高度 if + `#` 锚点）。
7. B3 线性 `SegmentGraphPlan`，先把现有脚本包进 graph。
8. B4 graph runtime 接 `#` 锚点 / `->` / `[if ->]` 的最小行为。
9. B5/B6 穿插推进，不能等功能全部完成才还债。

## 6. 非目标

- 不在 Phase B 初期抽纯 `packages/core`；纯 core 仍晚于 reader-runtime-web。
- 不实现 Phase C 的完整 `SignalRegistry` 与游戏化实时 Segment。
- 不启动 v1.7 plugin hook 系统，只保持 metadata 与 registry 入口可扩展。
- 不把 Android Reader 集成进当前 runtime 构建链；只保证未来可消费稳定产物。
- 不在 Phase B 初期实现任意 `.kmd` include/import；跨 `.kmd` 只作为 dependency graph 与宏/对象引用设计问题推进。
- **`flow` 主语与异形排版**（`selection-model.md` 流游标、回转/文字树 demo）不进 Phase B：
  它依赖 LayoutPlanner 的游标级改造，独立成后续阶段，Phase B 只保证选区模型不堵死它。
- 裸文本匹配（内容选择器二期）不进 Phase B（brainstorm）。

## 7. Gate

每个工作包至少满足：

- `pnpm build`
- `pnpm test:parser`
- `pnpm language:check`
- 关键样例脚本人工回归
- 新语法必须有 parser fixture 或等价回归脚本

B3 之后需要补 execution/graph 回归脚本，不能继续只依赖 parser test。

## 8. Open Questions

语言收敛已裁决并移出本清单的旧问题：`state.*` vs `var.*` 双命名空间（D19：不存在双命名空间）；
`+` 并发链能否混写 stage/layout/effect（D10：从句让任何主语可进链，`+` 只连完整句子）；
文本插值形态（D23：`{=}`）；loop 与 wait 的表面语法（D25–D27：后向导向即循环、`pause(事件)`，
`@ loop`/`@ wait` 不再存在；循环条件即普通条件边，运行时求值，"N 表达式求值时机"问题随之消解）。

仍开放：

- graph seek 对非 default path 的 UI 预期：近似跳转、路径选择器，还是禁止直接 seek。
- legacy `TextPlayer.play()` 的删除条件：需要哪些样例和自动测试覆盖后才安全。
- `.kmd` 引用另一个 `.kmd` 时，是导入宏/对象、跳到外部 segment，还是二者都允许。
- 跨文档引用的作用域：文档级 var 是否共享，对象是否按文档隔离，循环引用如何诊断。
- Work 与 `.kmd` 依赖图的绑定方式：依赖是否属于 Work revision manifest，还是语言级 import map。
- 兼容形态（`f.`、`hold/ease` 单词形、`camZoom`）的弃用期长度与移除时机（migration 步骤 4）。
